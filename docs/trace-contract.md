# Trace Contract

Trace is the public audit layer for one runtime decision path. It records enough metadata for runtime accountability
without reproducing engine internals or private effects.

Runtime Trace records may include policy-approved metadata such as:

- `trace_id`
- `cycle_id`
- `engine_id`
- `engine_version`
- `contract_version`
- `input_snapshot_hash`
- `decision_id`
- `governance_flags`
- `governance_result`
- `fallback_used`
- `created_at`

Explanation fields should be disclosed only when the runtime policy allows them and they do not reveal private strategy,
samples, tuning data, or reproducible internal effects.

Trace records must not include:

- Private parameters.
- Internal test vectors.
- Non-public samples.
- Complete details that reproduce private effects.
- Secrets, API keys, connection strings, or authorization material.

Trace is generated by the Runtime. An Engine may provide limited explanation metadata, but it does not own the final
public audit record.

# Trace Contract

Trace is the public explanation layer for one cognitive run. It must explain which Engine was used, what input digest was evaluated, what decision was returned, what governance result was applied, and why the final result is valid.

Runtime Trace records include at least:

- `trace_id`
- `cycle_id`
- `engine_id`
- `engine_version`
- `contract_version`
- `input_snapshot_hash`
- `decision_id`
- `reason_codes`
- `confidence`
- `governance_flags`
- `governance_result`
- `fallback_used`
- `created_at`

Trace records must not include:

- Private parameters.
- Internal test vectors.
- Non-public samples.
- Complete details that reproduce private effects.
- Secrets, API keys, connection strings, or authorization material.

Trace is generated by the Runtime. An Engine may provide reason codes and confidence, but it does not own the final public audit record.
